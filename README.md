## 🧩 كيف يعمل المشروع

- `index.html` يحتوي على واجهة مستخدم كاملة مع أقسام الصفحة الرئيسية، المنيو، سلة الطلب، والتواصل.
- `script.js` يتحكم في جلب البيانات، عرض المنتجات، البحث، الفلترة، وتحديث السلة.
- `sw.js` يدير خدمة العامل (Service Worker) لتخزين الملفات مؤقتًا وتوفير تجربة أوفلاين.
- `register-sw.js` يسجل خدمة العامل بشكل آمن دون الحاجة إلى كتلة سكربت داخلية.
- `manifest.json` يجعل الموقع قابلًا للتثبيت كتطبيق ويب على الهاتف.
- `offline.html` صفحة احتياطية تظهر عندما يكون الاتصال غير متوفر.
- `db-seed.js` يستخدم لتهيئة بيانات تجريبية في المتصفح أو في مصدر بيانات مرتبط.
- `settings.json` يحتوي على إعدادات إضافية للمشروع يمكن تعديلها بسهولة.

### وصف الملفات

- `index.html` — الصفحة الرئيسية والتطبيق الرئيسي.
- `style.css` — تصميم الواجهة وتنسيق الألوان.
- `script.js` — المنطق البرمجي لإدارة المنيو والسلة والتصفح.
- `manifest.json` — بيانات التطبيق لتثبيت PWA.
- `sw.js` — خدمة العامل والتخزين المؤقت.
- `register-sw.js` — تسجيل آمن لخدمة العامل.
- `offline.html` — صفحة بديلة عند فقدان الاتصال.
- `db-seed.js` — تهيئة بيانات تجريبية للمنيو والمستخدمين.
- `icons/` — أيقونات التطبيق وملفات دعم PWA.
- `Code.gs` — سكربت Google Apps Script (إذا كان مرتبطًا بنظام البيانات).
- `settings.json` — إعدادات إضافية للمشروع.
- `QUICK_START.sh` و `START_HERE.sh` — سكربتات مساعدة لبدء المشروع.

## 🚀 التشغيل محليًا

لتشغيل المشروع محليًا، يجب استخدام خادم ويب لأن خدمة العامل لا تعمل عبر `file://`.

### مثال باستخدام Python

```powershell
cd "f:\My Order System"
python -m http.server 8000
```

ثم افتح الصفحة:

```text
http://localhost:8000
```

### خيار بديل: Live Server في VS Code

- افتح المجلد في VS Code
- اختر "Open with Live Server"
- تصفح المشروع مباشرة عبر متصفحك

## 🌐 النشر على GitHub Pages

1. أنشئ مستودعًا جديدًا على GitHub.
2. ارفع الملفّات كاملة.
3. في إعدادات المستودع اذهب إلى `Pages`.
4. اختر فرع `main` أو `master` والمجلد `/(root)`.
5. احفظ وانتظر التشغيل.
6. سيصبح الرابط:

   `https://<your-github-username>.github.io/<your-repo>/`
